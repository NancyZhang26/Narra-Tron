## 2. Abstract

Narra-Tron is an AI-assisted Text-To-Speech system paired with an automatic page-turning mechanism designed to make reading more accessible. Once a book is inserted into the cradle, the system captures an image of the page and plays the corresponding audio output generated by an LLM library through a speaker. These mechanisms run on the Raspberry Pi. Once both pages are read, motors connected to the Raspberry Pico spin in coordination to flip the page, and the process repeats.

Progress so far includes a working TTS proof-of-concept webapp, functional motor mounts with synchronized firing, initial camera interface with automated image capture and persistent file storage, and a Dockerized Linux development environment for cross-platform compatibility. The team has not encountered major blockers and expects to finish before the end of classes.

Future plans include merging all components into a unified protocol, completing the camera mount for optimal focus height, finalizing the physical cradle structure, and ensuring smooth page-to-page transitions for the live demo.


## 3. Project Overview

### 4.1 Project Description

The goal of Narra-Tron is to bring reading to be more accessible through an AI-assisted Text-To-Speech system paired with an automatic page-turning mechanism. Once a book has been inserted into the cradle, it will take a picture and have a speaker play the output file produced by the LLM library. These mechanisms belong on the Raspberry Pi. Once both pages are completed, the motors connected to the Raspberry Pico will spin in coordination to flip the page, and the process begins again.

## 5. Progress

### 5.1 Hardware Progress

Motor mounts for the 2 motors have been created, and a decently high precision of the motors firing in sync and correctly on time has been achieved. More manual testing is still needed to ensure near 100% accuracy. The proof of concept for the page-flipping mechanism is complete and confirmed viable. A mount for the camera still needs to be created so it sits above the book at the correct height without obstructions.

### 5.2 Software Progress

A webapp proof-of-concept has been developed that accepts a picture of a book page and outputs a `.wav` file using LLM TTS libraries. Once all protocols are established between the Pi and Pico, the HTML layer will be removed and only the core TTS logic will remain. The software expects images from the Pi to be placed in a specific directory with a specific naming convention before beginning the TTS process.

On the camera side, an initial interface has been created to automate image capture and persistent file storage. A Dockerfile with a Linux image was set up to ensure a consistent development environment across all team members, since some Pi dependencies are only available on Linux.

The Pico camera can now successfully take a scan and save it directly to a file for downstream processing. VSCode has been installed and configured on the Pico with the team's Git repository running.

### 5.3 Current Results

- TTS webapp proof-of-concept: functioning
- Motor synchronization proof-of-concept: functioning
- Camera image capture interface: functioning
- Docker development environment: set up 
- Unified protocol between Pi and Pico: in progress 


## 6. Challenges and Solutions

- **Pi library compatibility:** Some Raspberry Pi dependencies are only available on Linux. Resolved by setting up a Dockerfile with a Linux image to standardize the development environment across all platforms.
- **Camera image quality:** The Pico camera can be finicky with picture quality. The team is working on finding the correct height and stabilization for the camera mount to ensure clear captures.
- **Pico setup:** Required multiple updates and reconfigurations to get Python libraries working properly. Now resolved.
- **Protocol coordination:** Defining communication protocols between the Pi and Pico so every step in the flow executes in the correct order is an ongoing challenge being actively worked on.
